# UNIT 4

# Lesson Plan

## REST API Administration and Troubleshooting

---

# Course Information

**Course Outcome (CO4)**

Apply Informatica Intelligent Data Management Cloud (IDMC) REST APIs for administrative automation, retrieve and manage organizational resources, analyze log files, troubleshoot integration issues, and utilize monitoring resources to maintain enterprise cloud integration environments.

---

# Course Learning Objectives

After completing this unit, students will be able to:

1. Explain the architecture and principles of REST APIs.
2. Use Informatica Cloud REST APIs for administrative operations.
3. Retrieve runtime, audit, organization, schedule, and user information using Platform REST APIs.
4. Analyze IDMC log files for troubleshooting.
5. Diagnose and resolve common IDMC administration issues.

---

# Lesson Plan

| Session | Topics Covered | Hours | Teaching Method | CO |
|:-------:|----------------|:----:|-----------------|:--:|
| **1** | REST API Overview – What is REST API, REST Architecture, Informatica Cloud REST API, Advantages, Platform REST APIs, REST API Guidelines | **3** | Interactive Lecture + Demonstration | CO4 |
| **2** | REST API Versions – Login URL Format, Base URL Format, Platform REST API Version 2 & Version 3 Resources, Authentication Concepts | **3** | Lecture + Demonstration | CO4 |
| **3** | Administrative REST APIs – Runtime Environments, Audit Logs, Schedule Information, Organization Details, Sub-Organization Details, User Information | **3** | Lecture + Case Study | CO4 |
| **4** | Troubleshooting – Session Log, Error Log, Success Log, Tomcat Log, Log Naming Convention, Log File Storage, IICS Status Page | **3** | Lecture + Demonstration | CO4 |
| **5** | Troubleshooting Common Issues, Accessing IICS Resources, Enterprise Troubleshooting Workflow, Unit Review | **3** | Discussion + Case Study | CO4 |

**Total Contact Hours: 15**

---

# Session-wise Learning Outcomes

### Session 1

Students will be able to:

- Explain REST architecture.
- Identify REST API components.
- Describe Informatica Platform REST APIs.

---

### Session 2

Students will be able to:

- Differentiate Platform REST API Version 2 and Version 3.
- Explain Login URL and Base URL formats.
- Understand REST API authentication.

---

### Session 3

Students will be able to:

- Retrieve Runtime Environment information.
- Retrieve Audit Log information.
- Retrieve Schedule information.
- Retrieve Organization and User details.

---

### Session 4

Students will be able to:

- Interpret Session Logs.
- Analyze Error Logs.
- Interpret Success Logs.
- Analyze Tomcat Logs.
- Identify log storage locations.

---

### Session 5

Students will be able to:

- Troubleshoot common IDMC issues.
- Use the IICS Status Page.
- Access Informatica support resources.
- Apply REST APIs and log analysis to enterprise troubleshooting scenarios.

---

# Teaching Strategy

The unit should be delivered using a combination of:

- Interactive lectures
- Live demonstrations
- Enterprise case studies
- Guided discussions
- Real-world troubleshooting examples

# UNIT 4

# Classroom Activities

**Course Outcome:** CO4

Apply Informatica Intelligent Data Management Cloud (IDMC) REST APIs for administrative automation, retrieve and manage organizational resources, analyze log files, troubleshoot integration issues, and utilize monitoring resources to maintain enterprise cloud integration environments.

---

# Introduction

The following classroom activities are designed to strengthen students' understanding of REST APIs, administrative automation, log analysis, and troubleshooting in IDMC. These activities emphasize problem-solving, teamwork, and application of concepts in enterprise scenarios.

---

# Activity 1

## Understanding REST APIs

### Objective

Explain the role of REST APIs in enterprise cloud administration.

### Scenario

A company manually checks Runtime Environment status every morning.

### Tasks

- Identify the drawbacks of manual administration.
- Explain how REST APIs can automate this process.
- List the benefits of automation.

### Deliverable

Short report (1 page).

---

# Activity 2

## REST API Version Comparison

### Objective

Compare Platform REST API Version 2 and Version 3.

### Tasks

Prepare a comparison table covering:

- Features
- Performance
- Compatibility
- Security
- Recommended use cases

### Deliverable

Comparison chart.

---

# Activity 3

## Authentication Workflow

### Objective

Understand API authentication.

### Scenario

An application needs secure access to IDMC resources.

### Tasks

Draw the authentication workflow showing:

- Login request
- Authentication
- Session token
- Protected API request
- JSON response

### Deliverable

Workflow diagram with explanation.

# Activity 4

## Administrative API Mapping

### Objective

Identify suitable APIs for administrative tasks.

### Tasks

Match each administrative requirement with the appropriate REST API.

Examples:

- Runtime monitoring
- User information
- Audit reports
- Schedule verification
- Organization details

### Deliverable

API mapping table.

# Activity 5

## Enterprise Dashboard Design

### Objective

Design an administrative dashboard.

### Scenario

An administrator wants a single dashboard displaying:

- Runtime status
- Active users
- Schedules
- Audit events

### Tasks

Design a dashboard layout and identify which REST APIs provide each piece of information.

### Deliverable

Dashboard sketch.

---

# Activity 6

## Log Analysis

### Objective

Interpret IDMC log files.

### Scenario

A scheduled task has failed.

Students are provided with sample:

- Session Log
- Error Log
- Success Log

### Tasks

- Identify the probable cause.
- Recommend corrective actions.
- Explain which log was most useful.

### Deliverable

Troubleshooting report.

---

# Activity 7

## Troubleshooting Workflow

### Objective

Apply a systematic troubleshooting approach.

### Scenario

### Tasks

Prepare a troubleshooting flowchart covering:

- Initial checks
- Log analysis
- Configuration validation
- Root cause identification
- Resolution

### Deliverable

Troubleshooting workflow diagram.

# Activity 9

## Enterprise Case Discussion

### Objective

# Expected Learning Outcomes

After completing these activities, students will be able to:

- Explain REST API fundamentals.
- Compare API versions.
- Understand authentication workflows.
- Select appropriate administrative APIs.
- Design monitoring dashboards.
- Analyze log files.
- Apply troubleshooting methodologies.
- Recommend preventive maintenance strategies.
- Integrate REST APIs into enterprise administration.
